## VERSION COURTE (recommandée pour l'engagement)

---

🤖 Et si votre équipe de développement était entièrement composée d'agents IA ?

Je viens de lancer un projet qui me tient à cœur : construire mon portfolio en **Blazor .NET** — mais pas tout seul.

Je l'ai confié à **Squad**, un framework open source de Brady Gaster qui crée de vraies équipes d'agents IA dans votre projet Git.

Concrètement, j'ai 5 agents au travail :
→ **Lead** — analyse les besoins, planifie, coordonne
→ **Frontend** — construit les composants Blazor, le design
→ **Backend** — développe les services et la logique métier
→ **Tester** — écrit les tests bUnit, valide chaque feature
→ **Scribe** — documente tout, enregistre chaque décision

Ce qui change tout par rapport à un simple assistant IA ?

**La mémoire.** Chaque agent accumule son historique dans Git. Entre deux sessions, ils se souviennent de ce qu'ils ont fait, pourquoi, et ce qu'ils ont appris.

**Les rôles.** Chaque agent a une charte claire. Pas de conflit, pas de chevauchement. C'est de la vraie spécialisation.

**La traçabilité.** Toutes les décisions architecturales sont loggées dans `.squad/decisions.md`. Dans 6 mois, je saurais exactement pourquoi j'ai choisi MudBlazor plutôt que Bootstrap.

Mon rôle dans tout ça ? Je définis les objectifs, je valide, j'approuve. **Squad amplifie, il ne remplace pas.**

Le projet est open source dès le départ — l'idée c'est de documenter chaque étape pour que vous puissiez voir Squad en action sur un cas réel.

## VERSION LONGUE (article LinkedIn ou post détaillé)

---

🚀 J'ai commencé à construire mon portfolio avec une équipe de 5 agents IA. Voici comment ça marche.

Il y a quelques semaines, je cherchais un moyen de faire deux choses en même temps :
1. Construire un vrai portfolio technique en **Blazor .NET**
2. Comprendre en profondeur comment fonctionnent les **agents IA dans un contexte de développement réel**

C'est là que j'ai découvert **Squad**, un projet open source de Brady Gaster.

---

**Squad, c'est quoi exactement ?**

Ce n'est pas un simple chatbot ou un assistant qui répond à vos questions.

Squad crée une **équipe persistante d'agents IA** qui vivent dans votre projet Git, chacun avec un rôle précis :

• **Lead Agent** — Il lit le backlog, découpe les tâches, assigne le bon agent au bon problème
• **Frontend Agent** — Expert Blazor/CSS, construit les composants visuels
• **Backend Agent** — Développe les services, les modèles, la logique
• **Tester Agent** — Écrit les tests bUnit, valide que rien ne casse
• **Scribe Agent** — Documente tout, maintient les décisions architecturales

La grande différence avec ChatGPT ou Copilot en mode solo : **les agents se souviennent**. Leur historique est commité dans Git. Entre deux sessions de travail, ils retrouvent le contexte, les décisions passées, les leçons apprises.

---

**Ce que j'ai découvert en l'utilisant**

Trois choses m'ont vraiment surpris :

**1. La spécialisation change la qualité**
Quand vous demandez à un agent Frontend de construire un composant Blazor, il ne réfléchit pas à l'architecture backend en même temps. La séparation des préoccupations s'applique aussi aux agents IA.

**2. La documentation devient naturelle**
Le Scribe Agent enregistre automatiquement pourquoi chaque décision a été prise. Dans trois mois, je n'aurai pas besoin de me souvenir pourquoi j'ai choisi GitHub Pages plutôt qu'Azure — c'est dans `.squad/decisions.md`.

**3. Vous restez toujours aux commandes**
Squad est conçu pour que l'humain ait le dernier mot. Les agents proposent, planifient, développent — mais rien ne merge sans votre approbation. C'est la bonne approche.
